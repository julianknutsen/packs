# The Idea Wizard

Come up with your very best ideas for improving this project.

## Phase 1: Diverge

Generate a list of 30 ideas (brief one-liner for each). Cast a wide net.
Use these techniques to push beyond obvious suggestions:

- **Analogies:** What would this look like in a different domain?
- **Inversions:** What if we did the opposite of current approach?
- **Cross-domain combinations:** Combine concepts from unrelated fields.
- **Constraint relaxation:** What if X limitation didn't exist?
- **User perspective:** What would make a new user's life dramatically
  easier?

Do not filter during this phase. Volume over quality.

## Phase 2: Evaluate

Go through each idea systematically and critically evaluate it. Reject
the ones that are not excellent choices, with clear reasons for
rejection. Keep the ones that pass your scrutiny.

## Phase 3: Converge

For each idea that passed evaluation, explain in detail:

1. **What:** The idea as a concrete, specific, actionable plan with
   detailed code snippets where relevant.
2. **Why:** Why it would be a good improvement.
3. **Downsides:** Possible risks or drawbacks.
4. **Confidence:** How confident you are that it actually improves the
   project (0-100%).

Rank by (Impact x Confidence) / Effort.

## Phase 4: Implement

Implement the top ideas now. Start with the highest-ranked idea and
work down.
